Primero que nada muchas gracias por este proyecto, mi problema es el siguiente, estoy utilizando el plugin de navegación de 11ty, tengo un directorio principal con otros subdirectorios cada uno con su respectivas notas en markdown, dentro de cada uno tengo un archivo Jason con el mismo nombre de la carpeta para añadir frontmatter a cada una de las notas, por ejemplo: 

Mi directorio tiene la siguiente estructura
```
post/
  post-1.md
  post-2.md
  index.md
  ...
  subfolder/
    otro-post.md
    index.md
    ...
```

Y en cada uno tengo mi archivo de datos

```
// post.json
{
  "layout": "base.njk",
  "eleventyComputed": {
    "eleventyNavigation": {
      "key": "{{ title }}",
      "parent": "..."
    }
  }
}
```

Por alguna razón al utilizar el siguiente fragmento de código en uno de mis archivos no se muestra nada 

`{{ collections.all | eleventyNavigation | eleventyNavigationToHtml | safe }}`

Entre las pruebas que estaba haciendo me di cuenta que ocurría porque el plugin de navegación no está "recibiendo" correctamente las propiedades, pensé que era un problema relacionado a las propiedades computadas, hasta que intenté esto y me retracté:

```
<ul>
{%- for post in collections.post -%}
  <li>{{ post.data.title }} - {{ post.data.eleventyNavigation.key }}</li>
{%- endfor -%}
</ul>
```

Básicamente me retracté porque muestra justo lo que esperaría. 

Entonces en resumidas cuentas no sé qué ocurre, como mencioné anteriormente creo que es un problema con el plugin de navegación ya que no está obteniendo correctamente las propiedades necesarias cuando se calculan de forma computada, pero en el fondo sí se están comportando de forma correcta
