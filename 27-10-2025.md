# AppBin — Compartir previsualizaciones HTML (solo vista)

AppBin es una aplicación para compartir previsualizaciones de HTML mediante un enlace público, al estilo de CodePen, pero sin compartir el código fuente. Los usuarios suben o generan una vista previa y el sistema crea un enlace que muestra únicamente la previsualización (render) en un entorno sandbox seguro.

## Descripción
AppBin permite a usuarios crear y compartir únicamente la previsualización (render) de documentos HTML. El código fuente no se expone ni se comparte públicamente; solo el resultado renderizado es accesible mediante el enlace generado. Esto es útil para compartir demos visuales, prototipos o resultados sin revelar la implementación.

Casos de uso:
- Compartir prototipos visuales con clientes o compañeros.
- Enviar demos a equipos QA sin exponer el código.
- Embebidos en documentación o chats para mostrar cómo se ve algo.

## Características principales
- Crear previsualizaciones HTML y generar un enlace único para visualización.
- Opciones de expiración del enlace (ej. 1h, 24h, permanente).
- Enlaces protegidos por contraseña (opcional).
- Visualización en iframe sandbox con políticas de seguridad (CSP).
- Miniaturas y metadatos (título, descripción, autor, fecha).
- Estadísticas simples: número de vistas por enlace (opcional).
- API REST para crear y recuperar previsualizaciones.
- Control de eliminación (el autor puede eliminar enlaces).
- Integración opcional con almacenamiento externo (S3) para assets.

## Seguridad y privacidad
Dado que el servicio sirve HTML renderizado, es crítico mitigar riesgos de seguridad:
- Servir la previsualización en un iframe con atributo sandbox para impedir ejecuciones peligrosas (por ejemplo: sandbox="allow-scripts" según sea necesario).
- Aplicar Content-Security-Policy estricta en la respuesta.
- Sanitizar y/o filtrar HTML si expone funciones sensibles. Una práctica recomendable es permitir solo recursos estáticos y deshabilitar navegación entre sitios.
- Almacenar sólo lo necesario: si el objetivo es no guardar código, considera almacenar solo el HTML renderable o una imagen/preview en lugar del código fuente completo.
- Opciones para eliminar o caducar enlaces automáticamente.

## Uso

### Desde la interfaz web
- Abrir http://localhost:3000
- Crear una nueva previsualización pegando o generando contenido HTML y elegir opciones: expiración, contraseña, título.
- Guardar y obtener el enlace público para compartir.

### API — ejemplo
Estos endpoints son ejemplos. Ajusta según la implementación real del proyecto.

Crear una previsualización (POST):
```bash
curl -X POST https://appbin.example.com/api/previews \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Demo botón animado",
    "html": "<!doctype html><html><body><button>Hola</button></body></html>",
    "expires_in": 86400,
    "password": "opcional"
  }'
```
Respuesta esperada (ejemplo):
```json
{
  "id": "a1b2c3d4",
  "url": "https://appbin.example.com/p/a1b2c3d4",
  "expires_at": "2025-10-28T12:00:00Z"
}
```

Ver la previsualización
- Abrir el enlace devuelto: https://appbin.example.com/p/a1b2c3d4
- El contenido se sirve dentro de un iframe sandbox para mitigar riesgos.

Eliminar una previsualización (DELETE, autenticado o mediante token):
```bash
curl -X DELETE https://appbin.example.com/api/previews/a1b2c3d4 \
  -H "Authorization: Bearer <TOKEN>"
```
